## Gestion de Eventos (Organizadores)



* #### **RF-05:** El sistema debe permitir al organizador crear nuevos eventos en la plataforma.

  * #### **RF-05.1:** El sistema debe permitir al organizador configurar la disponibilidad y variedad de cupos mediante la creación de tickets.

  * #### **RF-05.2:** El sistema debe permitir definir tickets tanto de pago (con tipo y precio) como gratuitos.

  * #### **RF-05.2:** El sistema debe permitir establecer un límite numérico por tipo de ticket o configurarlo como cupo ilimitado.

* #### **RF-06:** El sistema debe permitir al organizador modificar y actualizar la información de sus eventos existentes.

* #### **RF-07:** El sistema debe permitir al organizador visualizar un listado centralizado con todos sus eventos creados.

* #### **RF-08:** El sistema debe permitir al organizador dar de baja (desactivar) un evento.

* #### **RF-09:** El sistema debe permitir al organizador consultar estadísticas de sus eventos, incluyendo el conteo de visitas únicas y el promedio de puntuación recibido en cada uno de sus cronogramas.
